# Логи
**Установка логирования**

---
Каждый компонент устанавливается, как ArgoCD Application. В манифестах определено несколько источников для удобной работы с values-файлами:
- *Helm*-чарты устанавливаются из официальных репозиториев;
- values-файл для каждого чарта берутся из этого репозитория по пути `05-logs/helm`.

*Opensearch* и *Opensearch-Dashboard* устанавливается из чарта-обёртки, т.к. helm-чарт *Opensearch-Dashboard* не поддерживает HTTPRoute. В получившемся чарте:

- *Opensearch* и *Opensearch-Dashboard* устанавливаются как зависимости
- Определён HTTPRoute для сервиса *Opensearch-Dashboard*

---
## Компоненты
- `01-app-kafka` - *Apache Kafka* - развязывает агентов на нодах и агрегаторов логов, позволяет спокойно, без потери логов обновлять конфиг и перезапускать поды агрегаторов 
- `02-app-fluent` - *FluentBit Collector* - агент на нодах, собирающий логи контейнеров и хоста;
- `03-app-vector` - *Vector* - агрегатор логов, позволяет гибко работать с логами, благодаря полноценному языку обработки эвентов
- `04-app-opensearch` - *Opensearch* и *Opensearch-Dashboard* - хранилище и дашборд для собранных логов
