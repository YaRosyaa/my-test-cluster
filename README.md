# Задача
Это мой тестовый Kubernetes-кластер, где я разворачиваю и настраиваю различные сервисы и технологии, связанные с контейнеризацией, оркестрацией, CI/CD, мониторингом, логированием, безопасностью и управлением инфраструктурой. 

Проект служит песочницей для получения практического опыта работы с экосистемой Kubernetes и сопутствующими инструментами.

Состав кластера и используемые технологии регулярно обновляются по мере изучения новых инструментов и best practice.

*Текущий стек: Ansible, Kubernetes, Cert-Manager, MetalLB, GatewayAPI (Traefik Gateway), GitLab CI, ArgoCD, Victoria Metrics + Grafana + различные экспортеры*
# Этапы
```bash
├───00-Ansible           # Установка зависимостей и развёртывание кластера Kubernetes
│   └───roles
│       ├───prepare-host # Установка зависимостей + подготовка к развёртыванию
│       ├───first-master # Инициализация кластера на 1 control-plane ноде
│       └───add-host     # Добавление остальных хостов в кластере
├───01-Networking        # Обеспечение внешнего доступа в кластер
│   ├───01-cert-manager  # Создание CA и сертификатов
│   ├───02-metallb       # Настройка сервиса Load Balanceer от MetalLB
│   ├───03-gateway       # Создание Gateway с добавлением сертификата
│   ├───04-routes        # создание сервиса приложения и маршрутов к нему
│   └───charts
│       └───traefik      # values для Helm-чарта traefik
├───02-App               # Манифесты деплоя веб-приложения
│   └───app-repo-mirror  # Зеркало репозиторияЮ устанавливаемого приложения
├───03-ArgoCD            # ArgoCD
└───04-monitoring        # Контур мониторинга кластера в виде ArgoCD Application
```
