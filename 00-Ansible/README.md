# Этапы
## `roles/prepare-host`
Установка зависимостей и подготовка к развёртыванию:
1. Установка пакетов зависимостей и утилит для облегчения администрирования
2. Выполнение требований для запуска кластера:
	1. Отключение *swap* в рантайме и отключение монтирования *swap*
	2. Подключение необходимых модулей и определение опций ядра
3. Установка и конфигурация *containerd*
4. Установка необходимых бинарников
	1. CNI-плагины
	2. *crictl* - для удобства администрирования
	3. *kubeadm*
	4. *kubelet*
	5. *kubectl*  - для удобства администрирования
5. Определение конфигурации kubelet, как systemd-сервиса

## `roles/first-master`
Инициализация кластера на 1 control-plane ноде:
1. Загрузка манифеста *kubeadm* из шаблона
2. Инициализация кластера и сохранение вывода
3. Создание симлинка на *kubeconfig* для рута
4. Копирование и установка манифестов *flannel*

## `roles/first-master`
Добавление остальных хостов в кластере:
- На ноде с инициализированным кластером:
	1. Получение уже выпущенного *bootstrap*-токена, либо генерация нового
	2. Получение хеша CA
	3. Получение ключа для других control-plane нод
- Но остальных нодах:
	1. Проверка, что нода уже не принадлежит к кластеру
	2. Добавление ноды в кластер, в соответствии с её типов (control-plane, worker)
